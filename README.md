# Getting started
- Clone the repo
  - `git clone git@github.com:nynvr/nynvr.io.git`
- Install Docker Desktop
  - It has both Docker and Docker Compose
- Run docker compose
  - `docker compose up --build`
  - This will run 3 services,
    - backend service
      - Django app
      - `http://0.0.0.0:8000/`
      - `http://0.0.0.0:8000/api/`
      - `http://0.0.0.0:8000/admin`
      - Default Django superuser: nynvr
      - Default Django superuser password: password
      - Default Django superuser auth token: f53c73f61a57935dfdd2384a9269c8c2793f0075
    - web service
      - Next.js app
      - `http://0.0.0.0:3000/`
    - db service
      - PostgreSQL

# Production deployment  - Backend / Web services
- Github Actions <> AWS Integration
  - We are using IAM Role, OIDC, AssumeRoleWithWebIdentity
    - [Use IAM roles to connect GitHub Actions to actions in AWS](https://aws.amazon.com/blogs/security/use-iam-roles-to-connect-github-actions-to-actions-in-aws/)
  - All nynvr organization repositories' Github Actions are within the scope of this IAM Role.
- Set VPC
- Set Security Groups
- Set AWS ECR
- Set AWS RDS
- Set ECS Cluster
- Set ECS Service
  - Step 1: Environment
    - Compute options: Launch type
    - Launch type: FARGATE
    - Platform version: Latest
  - Step 2: Deployment configuration
   - Application type: Service
   - Task definition family: nynvrio-task-{backend}
   - Service name: nynvrio-service-{backend}
   - Service type: Replica
   - Desired tasks: 1
- Set Task Definition (if need to be created)
  - Step 1: Task definition configuration'
    - Task definition name: nynvrio-task-{backend}
  - Step 2: Infrastructure requirements
    - [yes] AWS Fargate 
    - OS, Architecture, Network mode:
      - OS: Linux/X86_64
      - Network mode: awsvpc
    - Task size
      - cpu: 1
      - memory: 2
    - Task role: github-actions-nynvrio
    - Task execution role: github-actions-nynvrio
  - Step 3: Container 1
    - Name: nynvrio-image-{backend}-ghub-hash
    - Image uri: nynvrio-image-{backend}-uri
    - Essential container: yes
    - Port mappings
      - remove autogenerated - 80 - TCP - HTTP
    - Resource allocation limits
      - cpu: 1
      - gpu: 1
      - memory hard limit: default - 3
      - memory soft limit: default - 1: 
    - Logging collection
      - [no] Use log collection
